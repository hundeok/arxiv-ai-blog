FVAttn: 비디오 생성을 위한 런타임 부하 분산을 갖춘 적응형 희소 어텐션

> **원본 논문 정보**
> - **제목:** FVAttn: Adaptive Sparse Attention with Runtime Load Balancing for Video Generation
> - **저자:** Hao Liu, Chenghuan Huang, Ye Huang, Zhiying Wen, Hao Liu, Mohan Zhang, Chen Li, Ziyang Ma, Jing Lyu, Jiangsu Du
> - **발행일:** 2026-07-17
> - **PDF 링크:** [FVAttn: Adaptive Sparse Attention with Runtime Load Balancing for Video Generation](https://arxiv.org/pdf/2607.16190v1)


✨ 3줄 핵심 요약
*   비디오 생성 모델에서 발생하는 '희소 어텐션의 부하 불균형' 문제를 해결하는 시스템입니다.
*   런타임에 GPU 간 부하를 실시간으로 재분배하고, 남는 자원을 활용해 추가 연산을 수행하여 효율을 극대화합니다.
*   FlashAttention 대비 4.41배의 어텐션 속도 향상과 2.02~2.11배의 전체 추론 속도 개선을 달성했습니다.

📖 쉽게 풀어쓴 1분 핵심
비디오 생성 모델(Video DiT)은 긴 영상을 처리할 때 '어텐션'이라는 연산에 매우 많은 시간이 소요됩니다. 이를 해결하기 위해 필요한 부분만 골라내는 '희소 어텐션(Sparse Attention)' 기법이 쓰이지만, 각 GPU에 할당된 연산량이 제각각이라 특정 GPU가 끝날 때까지 모두가 기다려야 하는 병목 현상이 발생합니다. 

FVAttn은 이 문제를 '런타임 부하 분산(RLB)'으로 해결합니다. 작업이 한쪽으로 쏠리면 즉시 연산 일부를 다른 GPU로 옮기고, 여유가 생긴 GPU에는 추가 연산을 시켜 자원을 알뜰하게 씁니다. 결과적으로 전체적인 기다림 시간을 획기적으로 줄여 비디오 생성 속도를 높였습니다.

🎯 배경 및 문제점
*   **병목 현상:** 고해상도 비디오 생성 시 어텐션 연산이 전체 추론 시간의 74.1%(Wan2.2-14B 기준)를 차지합니다.
*   **부하 불균형:** 기존의 'Top-p 라우팅' 방식은 각 헤드별로 연산량을 유동적으로 조절하는데, 이 과정에서 GPU별로 작업 부하가 불균등해집니다.
*   **연산 지연:** 여러 GPU가 병렬로 작동할 때 가장 느린 GPU(Straggler)가 전체 시스템의 속도를 늦추는 문제가 발생합니다.

💡 핵심 기술 및 아키텍처
*   **런타임 부하 분산 (RLB):** 어텐션 헤드들 중 연산량이 많은 무거운 헤드를 P2P 통신을 통해 다른 GPU로 옮겨 최장 연산 경로(Critical Path)를 단축합니다.
*   **슬랙 기반 희소 증강 (SASA):** 부하 분산 후에도 여유가 있는(비임계) GPU에 고가치의 블록 연산을 추가로 할당하여, 대기 시간을 유효한 연산 시간으로 전환합니다.
*   **중첩 최적화 (Overlap/Opt.):** 스케줄링, 데이터 이동, 연산 과정을 겹쳐서 처리함으로써 오버헤드를 숨기고 성능을 높입니다.

📊 실험 결과 분석
*   **부하 균형:** Wan2.2 I2V 모델에서 평균 부하 불균형 지수를 1.34에서 1.08로 개선했습니다.
*   **속도 향상:** FlashAttention 대비 어텐션 연산 속도를 4.41배 높였으며, 전체 DiT 추론 속도는 2.02~2.11배 빨라졌습니다.
*   **품질 유지:** 속도 개선에도 불구하고 경쟁력 있는 비디오 생성 품질을 유지했습니다.

🌍 세상에 미치는 영향 & 실무 적용 사례
*   **실무적 의의:** 고비용의 비디오 생성 서비스를 운영하는 기업이 동일한 하드웨어 환경에서 훨씬 빠르게 더 많은 영상을 생성할 수 있게 합니다.
*   **한계점:** 희소성이 적은 짧은 시퀀스나 일반 이미지 생성 작업에서는 효과가 감소합니다.
*   **확장성:** 현재 Ulysses-style 병렬 처리에 최적화되어 있으나, 향후 Ring Attention 등 다양한 분산 처리 환경으로의 확장이 가능할 것으로 예상됩니다.
*   **하드웨어 의존성:** GPU 간 통신 대역폭이 충분한 환경에서 더 뛰어난 성능을 발휘합니다.
