코드 레스큐: 코딩 에이전트를 위한 예산 보정식 복구 라우팅

코딩 에이전트가 예산 범위 내에서 에러 발생 시 최적의 경로로 문제를 해결할 수 있도록 돕는 효율적인 복구 체계입니다.

코드레스크: 코딩 에이전트를 위한 예산 보정형 복구 라우팅

✨ 3줄 핵심 요약
* 코딩 에이전트의 실패 시 무조건적인 고성능 모델 호출(Escalation) 대신, 상황에 맞는 최적의 복구 경로를 선택하는 '복구 라우팅' 기술을 제안합니다.
* 'Conformal Risk Control(CRC)' 계층을 추가하여 모델 재학습 없이도 사용자 예산에 따라 유연하게 비용과 성능의 균형을 실시간으로 조절합니다.
* 실험 결과, 고성능 모델로만 해결할 때보다 적은 비용(평균 복구 비용의 35%)으로 동일하거나 더 높은 문제 해결률을 달성했습니다.

📖 쉽게 풀어쓴 1분 핵심
코딩 에이전트가 문제를 풀다 실패했을 때, 기존 시스템은 단순히 '비싼 모델'에게 일을 넘기는 방식을 사용했습니다. 하지만 때로는 코드의 오류를 수정(Reflect)하거나 새로운 계획으로 다시 시도(Replan)하는 것만으로도 충분히 해결 가능한 경우가 많습니다. '코드레스크(CodeRescue)'는 실패 상황을 분석하여 1) 수정, 2) 재시도, 3) 고성능 모델 호출 중 가장 저렴하고 효과적인 경로를 선택합니다. 여기에 예산 관리 장치(CRC)를 더해, 사용자가 정한 예산 한도 안에서 가장 똑똑하게 행동하도록 시스템을 제어합니다.

🎯 배경 및 문제점
* **고정된 대응의 한계:** 기존의 비용 인식 시스템은 모델의 실패를 '더 강력한 모델이 필요한 상황'으로만 간주하여 무조건적인 에스컬레이션(Escalation)을 수행해 자원을 낭비합니다.
* **불필요한 고비용:** 난이도가 너무 낮으면 간단한 수정으로 충분하고, 난이도가 너무 높으면 비싼 모델을 써도 성공률이 보장되지 않는 경우가 발생하는데, 이를 구분하지 못하는 문제가 있습니다.
* **유연성 부족:** 사용자의 예산 요구사항이 바뀔 때마다 라우터를 재학습하는 것은 비효율적입니다.

💡 핵심 기술 및 아키텍처
* **3단계 복구 라우터:** 문제 설명, 실행 결과, 에러 메시지를 입력받아 세 가지 행동 중 하나를 선택합니다.
    * **Reflect(반영):** 피드백을 활용해 기존 코드를 직접 수정.
    * **Replan(재계획):** 저가형 모델로 처음부터 다시 시도.
    * **Escalate(에스컬레이션):** 고성능 모델에게 문제 해결을 위임.
* **CRC(Conformal Risk Control) 계층:** 라우터 상단에 위치하여 실시간으로 비용 페널티를 계산합니다. 이를 통해 모델 재학습 없이도 '예산-성능' 곡선을 따라 유연하게 동작을 변경할 수 있습니다.

📊 실험 결과 분석
* **성능 우위:** GPT-5.4-NANO/GPT-5.4 설정에서, 제안된 시스템은 무조건 에스컬레이션을 수행하는 기준 모델보다 적은 비용(평균 복구 비용의 35%)을 쓰면서도 더 높은 해결률을 보였습니다.
* **비단조적(Non-monotone) 특성 발견:** 난이도에 따라 고성능 모델 호출이 반드시 성공률을 높이지 않는 구간(너무 쉽거나, 너무 어려운 문제)이 존재함을 확인했습니다. 이 경우 CRC는 자동으로 에스컬레이션을 억제하여 비용 대비 성능을 최적화했습니다.

🌍 세상에 미치는 영향 & 실무 적용 사례
* **경제적인 AI 개발:** 복잡한 소프트웨어 엔지니어링 에이전트를 운영하는 기업들이 인프라 비용을 획기적으로 절감할 수 있습니다.
* **맞춤형 서비스:** 동일한 모델을 사용하더라도 사용자마다 설정한 예산 범위 내에서 최선의 결과를 도출하는 서비스 제공이 가능해집니다.
* **실무 적용:** 소프트웨어 자동화, 코드베이스 유지보수, 자동 버그 수정 도구 등 실행 환경(컴파일러, 테스트 툴 등)이 존재하는 모든 AI 코딩 워크플로우에 즉시 적용 가능합니다.
