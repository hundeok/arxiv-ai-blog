### ✨ [3줄 핵심 요약]
1. **사전학습이 RL 성능을 결정한다:** 사전학습 모델의 손실(Validation Loss)은 사후 강화학습(RL)이 도달할 '최종 성능'을 결정하며, 사전학습 데이터 토큰 수는 RL의 '성장 속도(학습 기울기)'를 수학적으로 정비례하게 결정합니다.
2. **단순한 연마(Sharpening)가 아니다:** RL은 기존 SFT 모델의 똑똑한 부분만 예리하게 만드는 것이 아니라, 어려운 문제에서 완전히 묻혀 있던 정답을 발굴(Tail Discovery)해 내지만 동시에 오답 패턴 또한 강화(Wrong-Mode Amplification)하는 이중적인 메커니즘을 가집니다.
3. **투자 예산의 황금 비율 존재:** 전체 연산량(Compute Budget)이 커질수록, 사전학습보다 강화학습(RL)에 투자해야 하는 자원 비율(RL Share)이 점진적으로 증가해야(최대 28% 이상) 가장 효율적인 추론 모델을 얻을 수 있습니다.

### 📖 [쉽게 풀어쓴 1분 핵심]
> **"체스 천재를 키우기 위해 교과서 읽기(사전학습)와 실전 문제 풀기(강화학습)의 황금 비율을 찾아냈습니다."**

최근 인공지능 트렌드는 스스로 생각하고 고집스럽게 정답을 찾아내는 **'추론형 AI(Reasoning AI)'**입니다. 이러한 AI를 만드는 과정은 크게 두 단계로 나뉩니다.

1. **교과서 정독하기 (사전학습/Pretraining):** 세상의 수많은 책과 자료(데이터)를 읽으며 기본적인 지식과 지능을 채우는 과정입니다.
2. **실전 시험지 풀고 오답노트 정리하기 (강화학습/RL):** 스스로 문제를 풀고 맞으면 보상을, 틀리면 벌을 받으며 문제 해결력을 고도로 집중시키는 과정입니다.

기존에는 "그냥 좋은 책을 최대한 많이 읽히는 게 장땡이다"라는 파(사전학습 중시형)와, "스스로 부딪히며 배우는 실전 학습이 최고다"라는 파(강화학습 중시형)가 갈려 있었습니다.

**이 논문은 이 둘 사이의 놀라운 비밀을 밝혀냅니다.**
아이의 **교과서 이해도(사전학습 손실)**를 보면 실전 문제풀이 학습을 시켰을 때 **최종적으로 몇 점까지 갈지(도달 성능)**를 정확히 맞출 수 있습니다. 또한, **교과서를 얼마나 꼼꼼하게 다독했는지(사전학습 토큰 수)**는 실전 훈련을 할 때 **성적이 얼마나 빠르게 수직 상승하는지(학습 속도)**를 결정합니다. 

특히 쉬운 문제는 실전 훈련을 통해 원래 알던 정답을 더 확실하게 맞히게 되지만, 어려운 시험 문제의 경우 구석에 처박아두었던 기막힌 아이디어를 끄집어내어 맞히기도(Tail Discovery) 하지만, 잘못 짚은 삼천포(오답)로 빠지는 습관도 함께 강화됩니다. 결국, 한정된 공부 시간(연산 자원) 내에서 최고의 성적을 내려면 공부 기간이 길어질수록 '교과서 읽기'보다 '실전 오답노트 풀기'의 비율을 점차 늘려야 한다는 수학적 공식(Scaling Law)을 우리에게 제시해 줍니다.

### 🎯 [배경 및 문제점]
> **"사전학습과 강화학습을 따로만 분석하던 기존 연구의 장벽을 허물고, 둘 사이의 거대한 연결고리를 추적합니다."**

현재 LLM 개발 파이프라인은 **[대규모 사전학습(Pretraining) ➔ 지도 미세조정(SFT) ➔ 강화학습(RL)]** 순서로 견고하게 자리 잡았습니다. 그러나 OpenAI의 o1, DeepSeek-R1 등 추론 모델들이 등장하면서 연산 비용(Compute)을 어디에 집중적으로 배치해야 하는가에 대한 엔지니어들의 논쟁이 촉발되었습니다.

* **사전학습 우선론 (Prior-centric):** 모델 크기와 인간이 작성한 텍스트 데이터셋을 극한으로 늘려 베이스 모델 자체의 역량을 키우는 것이 최우선이라는 시각입니다.
* **경험 우선론 (Experience-centric):** AlphaZero의 사례처럼, 스스로 환경과 상호작용하고 결과 기반의 피드백(RL)을 통해 인간의 모방을 넘어선 새로운 스킬을 발견해야 한다는 시각입니다.

#### 기존 연구의 한계점
1. **제어 불가능한 데이터 혼합:** 프론티어 급 LLM의 사전학습 말뭉치는 너무나 거대하고 불투명하여 모델의 어떤 능력이 사전학습에서 기인했는지, 사후학습(SFT/RL)에서 발현되었는지 과학적으로 추적하기가 불가능에 가까웠습니다.
2. **천문학적인 비용 장벽:** 사전학습과 사후학습 연산량 모두를 정교하게 쓸어내리며(Compute Sweep) 그 둘 사이의 상호작용 법칙을 도출하는 실험은 일반적인 연구진에게는 비용적으로 감당할 수 없었습니다.
3. **블랙박스 추론 경로:** 자연어 기반 추론 평가는 대개 최종 답변의 정답 유무만 측정할 뿐, 모델의 내부 정책(Policy)과 개별 토큰 레벨의 추론 경로(CoT)가 어떻게 변화하는지 정밀하게 분석하기 어렵습니다.

이 문제를 해결하기 위해 본 논문 연구진은 **체스(Chess)**를 완벽하게 통제된 실험 환경(Testbed)으로 채택하여, **5M(500만)부터 1B(10억) 파라미터**에 이르는 총 10개 규모의 모델들로 총 36개의 사전학습-RL 조합을 스윕하는 유례없는 실험을 진행하였습니다.

> **"체스판 위에서 구현된 LLM 파이프라인과 GRPO, 그리고 수학 영역으로의 성공적인 확장 아키텍처를 파헤칩니다."**

#### 1. 체스 기반의 AI 추론 실험 모델 아키텍처 (Chess Testbed)
이 연구는 체스를 일반적인 자연어처럼 토큰화하여 LLM 학습 구조와 완전히 일치시켰습니다.

* **체스 표현식(Chess Representation):** 각 기물의 움직임을 4개의 토큰 `⟨기물⟩; ⟨출발지⟩; ⟨목적지⟩; ⟨특수플래그(체크, 프로모션 등)⟩`으로 직렬화했습니다. 단어 사전(Vocabulary) 크기는 $|V| = 81$입니다.
* **모델 아키텍처:** 밀집(Dense) 디코더 전용 트랜스포머인 **Qwen3 아키텍처**를 기반으로 10가지 크기 모델을 설계하여 깊이감 있는 파라미터 스케일을 확보했습니다.

| 모델 크기 | 레이어 수 | Hidden Size | Intermediate Size | Heads (Q/KV) |
|:---:|:---:|:---:|:---:|:---:|
| **5M** | 6 | 256 | 768 | 2 / 2 |
| **50M** | 12 | 512 | 1536 | 8 / 4 |
| **200M** | 24 | 768 | 2304 | 12 / 4 |
| **1B** | 32 | 1536 | 4608 | 24 / 4 |

#### 2. 3단계 파이프라인 구현 상세

```
[1단계: 사전학습]
  Lichess 인간 대국 데이터 (54B 토큰) 학습 ➔ 다음 토큰 예측 객관식 훈련
       ▼
[2단계: SFT (생성적 사고 사슬 적용)]
  나무 구조 탐색 결과를 깊이 우선 탐색(DFS) 방식으로 직렬화하여 사고 사슬(CoT) 데이터 생성
  포맷: <T> [예상 경로 1] <sep> [예상 경로 2] </T> [최종 정답 기보]
       ▼
[3단계: 강화학습]
  환경 피드백 검증을 통해 오직 100% 정답 시 보상(=1) 지급하는 아웃컴 보상 환경 구축
  GRPO(Group Relative Policy Optimization) 알고리즘으로 정책 고도화
```

* **사전학습 (Pretraining):** Lichess의 블리츠/래피드 대국 기록 540억(54B) 토큰을 기반으로, 인간의 플레이 스타일 분포를 다음 토큰 예측(Next-Token Prediction) 방식으로 사전학습 시켰습니다.
* **지도 미세조정 (SFT) with Synthetic CoT:** 자연어 LLM이 생각하는 논리 구조(Chain-of-Thought)를 모사하기 위해, 기보 뒤에 정답만 학습하는 것이 아니라 **사고 사슬(CoT)** 데이터를 생성해 학습시켰습니다.
  * *방법론:* 사전학습 모델로부터 여러 개의 후보 수 행렬을 샘플링한 후, 접두사를 공유하는 트리 구조로 묶어 이를 DFS(깊이 우선 탐색) 순서로 직렬화하여 생성했습니다.
  * *포맷 예시:* `<T> 1. Rd8d2 Qc6e8+ <sep> 2. Kg8h7 </T> [최종 선택 수]` 형태로 인컨텍스트(In-context) 추론 유도를 구현했습니다.
* **강화학습 (RL) with Verifiable Rewards:** SFT를 마친 모델을 실제 퍼즐 환경에 얹고 **GRPO(Group Relative Policy Optimization)** 알고리즘을 사용해 학습시켰습니다.
  * 오답이 하나라도 섞이면 보상 `0`을 주며, 완벽한 정답 기보에 도달했을 때만 보상 `1`을 지급하는 아주 짜디짠 검증 가능 환경(Verifiable Reward Environment)에서 정책을 갱신시켰습니다.

### 📊 [실험 결과 분석]
> **"수학적 법칙으로 입증된 사전학습-RL 스케일링 법칙과 RL이 모델 내부를 바꾸는 기하학적 메커니즘을 공개합니다."**

#### 1. 공동 사전학습-강화학습 스케일링 법칙 (Joint Pretraining-RL Scaling Law)

본 논문의 핵심 기여는 사전학습과 강화학습 연산량 간의 복잡한 작용을 아래와 같은 **단 하나의 예측 방정식**으로 정립했다는 점입니다.

$$R(C_{RL}, N, T) = f(L_{pt}(N, T)) + g(N, T) \left( \log_{10} C_{RL} - \log_{10} C_{ref} \right)$$

* $R(C_{RL}, N, T)$는 강화학습 연산량 $C_{RL}$, 모델 파라미터 $N$, 사전학습 토큰 수 $T$ 하에서의 최종 보상(Pass@1 성능)입니다.
* $f(L_{pt})$는 기준이 되는 강화학습 연산량 $C_{ref} = 10^{20}$ FLOPs 지점에서의 보상 값으로, 오직 사전학습 검증 손실 $L_{pt}$에 의해서만 결정됩니다. (스피어만 상관계수 $\rho \approx -0.99$로 거의 완벽하게 반비례함.)
* $g(N, T)$는 강화학습 진행에 따른 **학습 속도(기울기)**를 나타내며, 사전학습 토큰 수 $T$에 크게 비례합니다.

$$g(N, T) \approx -0.208 + 0.017 \log_{10} T + 0.009 \log_{10} N$$

이 공식에 따르면, **더 많은 데이터를 먹인 사전학습 모델일수록 강화학습 시 학습 효율(기울기)이 기하급수적으로 빨라진다**는 것을 증명합니다.

```
       [사전학습 검증 손실 (Lpt)과 RL 성능의 관계]
       
  최종 RL 성능 (Pass@1)
     ▲
  0.4│                                 ● (사전학습이 잘된 체크포인트)
  0.3│                           ●
  0.2│                     ●
  0.1│               ● (사전학습이 덜된 체크포인트)
     └────────────────────────────────────────►
     낮음 ◀───────────────────────────────▶ 높음
                  사전학습 검증 손실 (Lpt)
```

#### 2. 파레토 최적 연산 자원 배분선 (Compute-Optimal Frontier)
한정된 총 FLOPs 예산 내에서 사전학습과 강화학습 자원 분배 최적점을 분석한 결과 다음과 같은 결론을 얻었습니다.

* **예산이 적을 때:** 사전학습 자원에 몰빵하는 것이 정답입니다. 충분히 다져지지 않은 모델은 강화학습 초기 효과를 누릴 수 없습니다.
* **예산이 충분히 클 때:** 전체 예산 중 **강화학습(RL)에 할당해야 하는 비율을 점차 늘려야** 합니다. (예: 50M 모델 최적 RL 비중 약 20% ➔ 680M 모델 최적 RL 비중 약 **28%**로 증가).

```
         [전체 연산량 대비 최적 RL 자원 할당 비율]
  
  최적 RL 할당 비중 (%)
    ▲
  30│                                         ★ (680M 규모 / 28% 할당)
  25│                        ★ (200M 규모 / 22% 할당)
  20│       ★ (50M 규모 / 20% 할당)
    └────────────────────────────────────────►
    10^17                 10^18                 10^19
                    전체 연산 예산 (FLOPs)
```

#### 3. RL 메커니즘 분석: RL은 단지 선명화(Sharpening) 작업일까?

이 연구는 "RL은 단지 SFT 모델이 선호하던 기보 중 정답 비율만 극대화하는 온도 조절(Sharpening) 장치에 불과하다"라는 일각의 주장을 철저히 검증하고, 그것이 반만 맞고 반은 틀렸음을 입증했습니다.

```
[쉬운 문제 (Easy Puzzles)] ➔ '정답 증폭 (Ground-Truth Amplification)'
SFT 상태에서 이미 알고 있던 정답 움직임의 확률을 100%에 가깝게 끌어올려 확실히 굳힙니다.

[어려운 문제 (Hard Puzzles)] ➔ '꼬리 발견 (Tail Discovery)' + '오답 증폭 (Wrong-Mode Amp)'
SFT 상태에서 극히 낮은 확률(<5%)로 존재하던 기막힌 정답 수를 표면 위로 건져 올립니다(Tail Discovery).
하지만 동시에, 매력적이지만 치명적인 오답 경로 역시 강하게 추종하여 오답 구렁텅이에 갇힙니다(Wrong-Mode Amplification).
```

이 현상은 **"왜 RL을 돌리면 Pass@1 성능(단 한 번 만에 정답을 맞출 확률)은 엄청나게 올라가는데, Pass@16 성능(16번 뽑아서 그중 하나라도 맞을 확률)은 거의 늘지 않거나 오히려 떨어지는가?"**에 대한 완벽한 수학적 배경을 제시합니다. RL이 어려운 문제에서 유연한 탐색 범위를 넓히는 게 아니라 특정 오답 모드로 빠르게 굳어버리게 만들기 때문입니다.

#### 4. 수학 영역(Math)으로의 확장 실험 검증

연구진은 체스에서 얻은 스케일링 법칙이 자연어 텍스트에서도 작동하는지 검증하기 위해 **1B 규모의 OLMo-2 모델**을 10B~200B 토큰의 수학 학습 환경(Math Domain)에서 훈련시켰습니다.

* **결과:** GSM8K 및 MATH500 벤치마크 테스트 결과, 사전학습 기간이 길어질수록(토큰 수 증가) 사후 RL 적용 시 도달 성능이 강력하게 수직 상승하고 학습 기울기가 가팔라짐을 확인하였습니다. 체스 실험실에서 도출한 이론이 실제 실무 자연어 추론 모델 설계에도 완벽히 적용되는 셈입니다.

### 🌍 [세상에 미치는 영향 & 실무 적용 사례]
> **"한정된 예산으로 최고의 AI 모델을 만들고 싶은 기업과 개발자를 위한 실전 엔지니어링 가이드입니다."**

본 연구의 디렉션은 현업의 생성형 AI 도입 팀과 엔지니어들에게 구체적인 액션 플랜을 제시합니다.

#### 1. '가성비' 높은 도메인 특화 추론 모델 빌딩 가이드
* **실무 문제:** 커스텀 코딩 에이전트나 금융 추론용 전용 모델을 학습시키려고 할 때, "데이터셋을 사서 사전학습을 더 돌릴 것인가? 아니면 빠르게 끝내고 RL을 빡빡하게 돌릴 것인가?"의 고민에 부딪힙니다.
* **해결책 (실무 룰):**
  * 베이스 모델의 손실 값이 일정 임계치(기초 상식 선)에 도달할 때까지는 사전학습에 전폭 투자하십시오.
  * 기초 지능이 잘 안착했다면, 전체 학습 예산의 **약 20%~30%는 무조건 사후 강화학습(RL) 전용 서버 자원**으로 남겨두는 배분식을 설계하십시오. 예산이 풍부할수록 RL 비중을 더욱 확대(최대 30% 수준)하는 것이 연산 비용 최적화(Compute-Optimal)의 지름길입니다.

#### 2. Pass@k 병목 현상 돌파를 위한 오답 모드 억제(Anti-Wrong-Mode) 기법 도입
* **실무 문제:** RL 적용 후 특정 패턴의 오답 에러가 고착화되며, 추론 다양성을 훼손하는 '오답 증폭' 현상이 나타납니다.
* **해결책:** 
  * 단순히 "맞췄다/틀렸다" 형태의 바이너리 보상(Outcome-based Reward)만 주면 오답 루프에 오염됩니다.
  * 추론의 중간 과정마다 보상을 개입시키는 **단계별 보상 모델(PRM, Process-supervised Reward Model)**을 설계하거나, RL 학습 루프 중에 주기적으로 사전학습 텍스트 또는 고품질 SFT 데이터를 함께 섞어서 학습시키는 **Interleaving(교차 학습)** 방식을 채택하여 모델 정책이 좁은 영역에 갇히는 현상을 적극적으로 방지해야 합니다.

#### 3. LLM의 효율적 추론 트리(DFS Serialization) 응용
* **실무 문제:** 추론형 에이전트(Agent) 구축 시 한도 끝도 없이 길어지는 CoT 토큰 소모량과 비효율적인 깊이 탐색 문제를 해결해야 합니다.
* **해결책:** 본 연구에서 개발된 'DFS 기반 트리 기보 직렬화 포맷'을 자연어 태스크에 응용할 수 있습니다. 
  * 에이전트가 단 하나의 정답 경로만 계속 쓰게 하지 말고, SFT 데이터 빌딩 과정에서 **"A안 탐색 결과 실패 ➔ B안 탐색 결과 성공 ➔ 따라서 B안 최종 확정"** 같은 탐색 및 백트래킹(Backtracking)의 로그 구조가 고스란히 담긴 합성 데이터를 사전에 대거 수집하여 SFT 시키십시오. 이 구조가 안착되어야만 소규모 경량 파라미터(예: 1B, 3B, 7B) 모델도 70B 이상의 거대 모델에 필적하는 고도의 자가 수정(Self-Correction) 능력을 지닐 수 있게 됩니다.
