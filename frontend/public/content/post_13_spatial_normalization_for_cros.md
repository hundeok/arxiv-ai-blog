### 🎯 [배경 및 문제점]

> 💡 **의료 기기와 환자마다 제각각인 망막 데이터 구조가 AI의 학습과 실전 추론 성능을 크게 저하시킵니다.**

빛을 쏘아 망막의 단면을 촬영하는 **안과 광간섭 단층촬영(OCT, Optical Coherence Tomography)**은 안과 질환뿐만 아니라 다발성 경화증(MS), 알츠하이머(AD), 파킨슨병(PK) 등 신경변성 질환의 바이오마커를 추출하는 핵심 기술입니다. 특히 황반 영역의 신경절세포층-내망상층(GCL+IPL) 두께 변화는 뇌 신경계의 손상을 간접적으로 보여주는 매우 중요한 지표입니다.

#### 1. 기존 기술의 한계
* **이미지 레벨의 한계**: speckle 노이즈, 혈관으로 인한 그림자 왜곡(Shadowing Artifacts), 얇은 레이어 간의 희미한 명암 대비는 경계선 획정을 어렵게 만듭니다.
* **해부학적 및 기기 편차**: 사람마다 망막의 곡률과 두께가 다를 뿐만 아니라, 장비 제조사나 촬영 프로토콜(FastMac, PPole 등)에 따라 데이터의 기하학적 형태가 완전히 달라집니다. 이를 **기하학적 도메인 시프트(Geometric Domain Shift)**라고 합니다.
* **레이블 데이터의 부족**: 8개나 되는 얇은 망막 레이어 경계면을 숙련된 의사가 한 땀 한 땀 레이블링하는 것은 극도로 정성이 많이 들어가는 작업입니다. 따라서 학습 데이터가 항상 부족합니다.

#### 2. 일반화(Generalization)의 참패
기존 딥러닝 모델들은 특정 병원의 특정 장비로 찍은 데이터셋(예: Johns Hopkins 대학 데이터)에서는 높은 성능을 보이지만, 다른 병원의 장비(예: Miguel Servet 병원 데이터)에 적용하면 기하학적 왜곡을 극복하지 못해 경계선이 서로 꼬이거나 끊어지는 해부학적 오류를 범하게 됩니다.

### 💡 [핵심 기술 및 아키텍처]

> 💡 **황반 중심의 3D 회전 변환 알고리즘과 위상 제약 기반의 네트워크 설계로 도메인 간 물리 장벽을 허뭅니다.**

#### 1. 황반 중심 공간 정규화(Fovea-Centered Spatial Normalization) 파이프라인
연구진이 개발한 전처리 프로세스는 수학적으로 매우 엄밀하면서도 명확한 단계로 구성됩니다.

```
[입력 데이터] ──> [중앙 B-scan 검출] ──> [ILM/RPE 대략적 분할] ──> [최소 두께 지점(황반, FP) 정의]
                                                                                │
[정규화 완료] <── [일관성 크롭핑(Cropping)] <── [RPE 최소자승법 기반 수평 회전] <┘
```

* **황반점(FP, Foveal Point) 위치 탐색**: 내경계막(ILM)과 망막색소상피(RPE) 간의 물리적 거리가 가장 짧은 위치(두께가 가장 얇은 곡선의 최저점)를 황반점으로 정의하고, 이 점을 3차원 볼륨의 정중앙($[x, y] = [0, 0]$)에 오도록 평행 이동(Translation) 시킵니다.
* **RPE 수평 회전 정렬**: 황반 부근의 RPE 곡선을 최소자승법(Least Squares Approximation)으로 선형 피팅하여 기울기를 계산한 후, 3D 볼륨 전체를 역회전시켜 RPE 레이어가 완벽히 수평축에 평행하도록 만듭니다.
* **일관성 크롭핑(Consistency Cropping)**: 회전 후 공백이 생기거나 장비별 스캔 범위 차이로 발생하는 도메인 격차를 없애기 위해 표준 규격으로 이미지를 잘라냅니다.

| 데이터셋 | 오리지널 해상도 ($H \times W \times D$) | 공간 정규화 적용 후 해상도 |
| :--- | :--- | :--- |
| **JHopkins (FastMac)** | $496 \times 1024 \times 49$ | **$496 \times 900 \times 49$** |
| **MServet (FastMac)** | $496 \times 512 \times 25$ | **$496 \times 400 \times 25$** |
| **MServet (PPole)** | $496 \times 765 \times 61$ | **$496 \times 600 \times 61$** |

#### 2. 비교 분석된 SOTA 모델 3종 아키텍처 딥다이브

##### ① nnU-Net (No-New-UNet)
* **특징**: 네트워크 자체 구조 변경보다는 타깃 데이터에 맞춰 학습 파이프라인(배치 크기, 정규화, 데이터 증강 등)을 자동 최적화하는 프레임워크입니다.
* **한계**: 기하학적 변화에 수동적으로 대응하므로, 전처리(공간 정규화)가 갖춰지지 않으면 교차 도메인 상황에서 극도로 불안정합니다.

##### ② MGU-Net (Multi-scale Graph Convolutional Network-assisted UNet)
* **특징**: 인코더와 디코더 사이에 '멀티스케일 전역 추론 모듈(MGRM)'을 삽입하여, 픽셀 단위 정보뿐만 아니라 레이어 간의 상대적 배치 관계를 그래프 뉴럴 네트워크(GCN)로 학습합니다.
* **한계**: 학습 안정성이 다소 떨어져 수렴이 어렵고, 위상적 일관성을 완벽히 강제하지는 못합니다.

##### ③ SD-LayerNet (Supervised/Semi-supervised Disentangled LayerNet)
* **특징**: 본 논문에서 강력한 성능을 보여준 모델입니다. 물리적 공간 인자(Anatomical factor)와 이미지 스타일 인자(Modality factor)를 완전히 분리하여 학습하는 **Disentangled Representation** 기법을 적용했습니다.

```
                  ┌──> [공간/구조 인자] ──> [Supervised Segmentation] ──┐
[오리지널 이미지] ─┤                                                      ├──> [재구성 오차 복원]
                  └──> [스타일 인자] ────────────────────────────────────┘
```

* **해부학적 사전 제약(Anatomical Priors)**: 레이어의 두께 편차 제한, 절대 섞이거나 뒤바뀔 수 없는 적층형 위상 제약 조건(Topological Constraints)을 손실 함수(Loss)에 직접 반영하여, 레이어가 끊어지거나 위아래 순서가 뒤집히는 물리적 오류를 원천 차단합니다.

#### 2. 위상 오류 검증 (Topology Violations) 결과

#### 1. 의료 영상 분석 스타트업 및 연구소 가이드
* **콜드 스타트(Cold-Start) 극복**: 의료 데이터 구축에 수억 원을 들여 수천 장을 확보할 필요가 없습니다. 본 논문에서 제시한 것처럼 **황반 중심 전처리 파이프라인**을 선제적으로 구축하면, 단 수십 장의 정밀 주석 데이터만으로도 고품질 글로벌 모델 학습이 가능해집니다.
* **PACS 솔루션 통합**: DICOM 데이터를 파이썬 서버로 수신한 즉시 최소자승법 기반 RPE 수평 정렬을 거치게 하는 **전처리 미들웨어(Middleware)**를 작성해 배포하면 기기 호환성 이슈를 완벽히 격리할 수 있습니다.

#### 2. 정밀 분석 연구 및 바이오마커 발굴
* **고해상도 en-face 두께 맵 복원**: 기존 장비(Heidelberg 등)가 제공하는 대략적인 8x8 그리드 요약 정보 대신, **3차원 픽셀 단위 고해상도 두께 지도(Dense Representation Map)**를 실시간으로 그릴 수 있습니다. 이는 제약 회사에서 신경계 신약 유효성을 평가할 때 매우 정밀한 신체 반응 바이오마커로 사용될 수 있습니다.

```python
# [실무 적용을 위한 Fovea 중심 정렬 의사 코드]
import numpy as np
from scipy.optimize import curve_fit

def spatial_normalization_3d(oct_volume, ilm_curve, rpe_curve):
    # Step 1: 최소 두께 지점인 황반점(FP) 찾기
    thickness = rpe_curve - ilm_curve
    foveal_x = np.argmin(thickness)
    
    # Step 2: FP를 원점으로 평행 이동
    translated_volume = shift_volume_x(oct_volume, shift_amount=-foveal_x)
    
    # Step 3: FP 주변 RPE 곡선을 최소자승법으로 피팅하여 기울기(각도) 계산
    x_range = np.arange(foveal_x - 20, foveal_x + 20)
    slope, intercept = np.polyfit(x_range, rpe_curve[x_range], 1)
    angle_rad = np.arctan(slope)
    
    # Step 4: 3D Volume 전체를 역회전하여 수평 정렬 수행
    normalized_volume = rotate_volume_z(translated_volume, angle=-angle_rad)
    
    return normalized_volume
```
