# "수학을 못해요" 대신 "단위 변환이 약해요"라고 콕 집어주는 AI 치료법, CRAFT 논문 완전 정복

*   **저자**: Vipul Gupta, Zihao Wang, Razvan-Gabriel Dumitru, MohammadHossein Rezaei, Aakash Sabharwal, Yunzhong He (Scale AI)
*   **발행일**: 2026년 7월 17일

### ✨ [3줄 핵심 요약]
1. **문제 정의**: 기존 LLM 평가는 단순히 '어디서(Where)' 틀렸는지만 알려줄 뿐, SFT(미세조정) 데이터 구축에 꼭 필요한 '왜(Why)' 실패했는지에 대한 구체적인 진단이 부족했습니다.
2. **해결책 (CRAFT)**: 루브릭(채점 기준) 데이터를 활용해 LLM의 약점을 계층적 나무 구조(Tree)로 시각화하고, 모델마다 서로 다른 깊이의 취약점을 동적으로 찾아내 맞춤형 학습 데이터를 생성합니다.
3. **연구 성과**: 금융/법률 전문 도메인 13개 벤치마크 평가에서 Qwen3, Gemma-3, Llama-3.1 등 오픈소스 모델들의 성능을 대폭 끌어올리며, 프롬프트 단위나 무작위 데이터 생성 방식보다 압도적으로 우수한 미세조정 효율을 증명했습니다.

### 📖 [쉽게 풀어쓴 1분 핵심]
> **💡 한 줄 요약**: 시험 점수표만 보고 무작위로 문제집을 푸는 대신, 오답 노트의 구체적인 '감점 이유'를 모아 부족한 개념만 콕 집어 훈련하는 스마트 오답 과외 선생님입니다.

학교에서 수학 시험을 본 영희가 60점을 받았습니다. 성적표에는 단순히 **'수학 60점 (취약)'**이라고만 적혀 있습니다. 영희는 수학의 어느 부분을 더 공부해야 할까요? 방정식일까요, 기하학일까요, 아니면 단순 연산 실수일까요? 정보가 너무 부족합니다.

이때 똑똑한 과외 선생님이 등장해 영희의 시험지를 한 장 한 장 뜯어봅니다.
*   "1번 문제: 식은 완벽히 세웠으나 **소수점 계산 실수**로 감점"
*   "5번 문제: 개념은 맞았으나 **마지막에 단위를 안 적어서(kg, m 등)** 감점"

선생님은 이 채점 기준(Rubric)들을 모아 분석한 뒤, **"영희는 수학 전반이 아니라 '소수점 연산'과 '단위 표기'가 약하구나!"**라는 정밀 진단을 내립니다. 그리고 딱 이 부분만 집중 보완할 수 있는 맞춤형 연습 문제 100개를 만들어 영희에게 풀게 합니다. 결과는? 다음 시험에서 영희는 100점을 받게 됩니다.

**CRAFT(Clustering Rubrics for Actionable Fine-Tuning)**가 LLM을 학습시키는 원리가 바로 이와 같습니다. LLM이 생성한 답변의 상세 채점 기준(루브릭)을 분석해 모델의 미세한 구멍(약점)을 찾고, 그 구멍을 메우기 위한 맞춤형 데이터셋을 정밀 조준해 만들어 냅니다.

### 🎯 [배경 및 문제점]
> **💡 한 줄 요약**: 기존의 평가는 "어디서" 틀렸는지만 보여줄 뿐, 모델을 고치기 위한 "무엇을, 어떻게" 학습시켜야 하는지는 알려주지 못했습니다.

현재 AI 학계와 산업계는 LLM의 성능을 측정하기 위해 MMLU, GSM8K 등 수많은 벤치마크 평가를 수행합니다. 하지만 이러한 평가는 대개 **"금융 도메인 점수: 55%"**와 같이 거시적인 점수(Category-level scores)로만 요약됩니다.

이러한 방식은 두 가지 치명적인 한계를 가집니다.

```
[기존 평가 방식의 한계]
"수학 50점" ───> (대체 어떤 데이터를 더 모아야 하지? 단순 수학 문제 1만 개 생성?) -> 비효율적인 SFT
```

1. **실패 원인의 불명확성 (Implicit Failure)**:
   수학 문제에서 오답이 나왔을 때, 그것이 공식을 잘못 선택한 것인지, 계산 오류인지, 혹은 최종 답변 포맷을 맞추지 못한 것인지 구분할 수 없습니다. 즉, 실패의 위치(Where)는 알지만 구체적인 역량 실패 원인(Why)은 알 수 없습니다.
2. **학습 데이터 생성의 막막함**:
   모델을 고치기 위한 Post-training(사후 학습) 단계에서 "금융 데이터가 부족하니 금융 관련 텍스트를 무작위로 더 생성해서 학습시키자"라는 식의 비효율적이고 방대한 데이터 수집에 의존하게 만듭니다.

최근 프롬프트 자체를 클러스터링하여 약점을 진단하는 `EvalTree` 같은 기법이 제안되었으나, 이 역시 프롬프트 전체 맥락에 매몰되어 **하나의 답변 요구사항 속에 숨겨진 여러 세부 역량(예: 법률 해석 능력 vs 법령 인용 포맷팅 능력)을 독립적으로 분리해 내지 못한다**는 한계가 있었습니다.

### 💡 [핵심 기술 및 아키텍처]
> **💡 한 줄 요약**: 루브릭 기준을 '역량 탐침기'로 전환하여 계층적 트리를 만들고, 모델이 유독 약한 마디(Node)를 동적으로 찾아내 저격수처럼 정밀 타격하는 학습 데이터를 설계합니다.

CRAFT의 전체 파이프라인은 크게 **4단계**로 구성됩니다.

```
[CRAFT Pipeline]
1. 루브릭 기반 데이터 입력 ──> 2. 세부 역량 기술서 추출 ──> 3. 계층적 트리 구축 및 동적 약점 노드 선정 ──> 4. 표적 SFT 데이터 생성 및 미세조정
```

#### 1단계: 루브릭으로부터 역량 정보 추출 (Extracting Capabilities)
CRAFT는 프롬프트와 루브릭 기준(Criterion)의 쌍을 분석하여, 해당 채점 기준을 통과하기 위해 모델에 요구되는 구체적인 역량을 탈맥락화된 짧은 텍스트(Capability Description)로 추출합니다.

*   *원문 프롬프트*: "부채 비율이 2.5배인 기업에 250bps 금리 충격이 가해졌을 때 자본 지출(Capex) 영향은?"
*   *루브릭 기준*: "답변은 유지보수 Capex와 성장 Capex의 차이를 명확히 구분해야 함."
*   *추출된 역량 기술*: **"전략적 자본 지출(Capex) 분류 및 배분 분석 능력"**

#### 2단계: 계층적 클러스터링 트리 구축 (Hierarchical Clustering)
추출된 수만 개의 역량 기술서들을 구조화된 트리로 조직화합니다.
*   **고정 상위 레벨 (Fixed Levels)**: 데이터셋이 가진 메타데이터(예: L0-금융/법률, L1-세부 카테고리)를 트리의 상단에 고정 배치하여 도메인이 뒤섞이는 것을 방지합니다.
*   **하위 레벨 자동 학습 (Learned Levels)**: 하위 노드는 임베딩 모델(`text-embedding-3-small`)을 통해 벡터화된 후 **k-means 알고리즘**으로 군집화됩니다. 이후 LLM(Gemini 2.5 Pro)이 군집들을 대표하는 더 넓은 개념의 상위 노드명을 생성하며, 데이터 크기에 맞게 자동으로 깊이(Depth)가 조정됩니다.

#### 3단계: 동적 약점 노드 탐색 (Dynamic Node Selection)
구축된 트리의 각 노드에 타깃 LLM의 실제 채점 결과(Pass/Fail)를 매핑합니다.
*   **왜 '동적(Dynamic)'인가?** 
    모델마다 약점의 깊이가 다릅니다. 초보 모델은 "금융 지식 부족"이라는 넓은 노드(상위 레벨) 전체가 약점일 수 있지만, 고성능 모델은 "국경 간 세금 회피 규정 해석" 같은 극히 좁은 노드(하위 레벨)에서만 실패할 수 있습니다. 
    CRAFT는 사전에 지정된 트리 깊이에 얽매이지 않고, 충분한 서포트 데이터(예: 평가 기준 30개 이상)가 확보된 노드 중 **합격률이 급격히 떨어지는 경계면을 위에서 아래로 탐색하며 동적으로 선택**합니다.

#### 4단계: 타깃 지향적 데이터 생성 및 SFT (Targeted Fine-Tuning)
선정된 약점 노드(예: "전략적 자본 지출 배분 분석")를 바탕으로 교사(Teacher) 모델을 사용해 맞춤형 프롬프트와 고품질 답변 세트를 노드당 약 40개씩 생성합니다. 이때 프롬프트 생성 시 주변 형제 노드(Sibling Nodes)들의 정보도 함께 컨텍스트로 제공하여, **이미 모델이 잘하는 영역과 데이터가 겹치지 않도록 방어막(Negative Constraints)을 칩니다.**

### 📊 [실험 결과 분석]
> **💡 한 줄 요약**: 경쟁 기술 대비 최소한의 표적 데이터(단 1,000개)만으로도 도메인 전반의 미세조정 성능을 최고 수준으로 끌어올렸습니다.

본 연구에서는 **Qwen3-4B, Qwen3-8B, Gemma-3-4B, Llama-3.1-8B-Instruct** 등 4가지 오픈소스 모델을 활용해 **금융(Finance)** 및 **법률(Legal)** 분야에서 실험을 진행했습니다. 진단용 데이터셋과 겹치지 않는 완전히 독립된 **13개 벤치마크**를 통해 평가의 신뢰성을 확보했습니다.

#### 1. 금융 도메인 평가 결과 (평균 정확도 %, 세부 수치)
금융 도메인 내 6개 벤치마크(FinanceBench, ConvFinQA, TAT-QA 등)의 평균 성능 비교입니다.

| 모델 | Random (무작위) | EvalTree (프롬프트 단위) | CRAFT (루브릭 단위, 제안 방식) |
| :--- | :---: | :---: | :---: |
| **Qwen3-4B** | 37.6% | 40.8% | **46.0%** (★ +5.2%p 우세) |
| **Qwen3-8B** | 44.1% | 43.5% | **45.1%** (★ +1.0%p 우세) |
| **Gemma-3-4B** | 38.4% | 38.6% | **39.8%** (★ +1.2%p 우세) |
| **Llama-3.1-8B** | 36.6% | 36.0% | **42.5%** (★ +5.9%p 우세) |

*   **해설**: 4개 모델 전체에서 CRAFT가 압도적 1위를 기록했습니다. 특히 Llama-3.1-8B 모델의 경우 무작위 학습 대비 **무려 5.9%p의 성능 향상**을 이루어냈습니다. 이는 타깃이 명확하지 않은 단순 도메인 데이터 주입(Random)이나 질문 단위 분류(EvalTree)보다, 구체적인 채점 기준 단위의 약점 보완이 훨씬 효과적임을 방증합니다.

#### 2. 법률 도메인 평가 결과 (평균 정확도 %)
법률 도메인 내 7개 벤치마크(LegalBench, CaseHOLD, SARA 등)의 평균 성능입니다.

| 모델 | Random (무작위) | EvalTree (프롬프트 단위) | CRAFT (루브릭 단위, 제안 방식) |
| :--- | :---: | :---: | :---: |
| **Qwen3-4B** | 49.9% | 51.7% | **53.0%** (★ +1.3%p 우세) |
| **Qwen3-8B** | 61.2% | **62.1%** | 61.5% (오차범위 내 안착) |
| **Gemma-3-4B** | 51.4% | 50.7% | **51.7%** (★ +0.3%p 우세) |
| **Llama-3.1-8B** | 46.9% | 48.2% | **51.0%** (★ +2.8%p 우세) |

*   **해설**: 법률 분야에서도 4개 모델 중 3개 모델에서 CRAFT가 가장 높은 성적을 냈습니다. Qwen3-8B 모델의 경우에만 EvalTree가 미세하게 앞섰으나, 디코딩 편차 범위 내에 존재하여 통계적으로 유의미한 격차는 아니었습니다.

#### 3. 핵심 아블레이션 연구: 동적(Dynamic) vs 고정(Fixed) 깊이 탐색
CRAFT의 '동적 트리 탐색' 알고리즘이 실제로 유효한지 검증하기 위해, 트리의 특정 깊이(L4 또는 L5)로 고정하여 약점을 추출한 후 학습시켰을 때의 결과와 비교했습니다.

```
[아블레이션 실험 결과 - 평균 정확도]
- Qwen3-4B (Legal)   : Fixed-L4 (47.8%) / Fixed-L5 (49.0%) / Dynamic (50.4% ★)
- Llama-3.1-8B (Legal): Fixed-L4 (43.6%) / Fixed-L5 (40.1%) / Dynamic (50.3% ★)
```

어떤 도메인에서는 미세한 약점(L5)을 고치는 것이 좋고, 다른 곳에서는 포괄적인 약점(L4)을 다루는 것이 유용하기 때문에 고정된 깊이는 성능의 기복을 야기합니다. 반면 **동적 선택(Dynamic)은 항상 최적의 깊이를 유연하게 찾아내어 모든 조건에서 가장 높은 성능을 일관되게 보여주었습니다.**

### 🌍 [세상에 미치는 영향 & 실무 적용 사례]
> **💡 한 줄 요약**: 무의미한 '합성 데이터 무한 복제'의 시대를 끝내고, 소량의 저비용 고효율 고품질 데이터셋만으로 기업 맞춤형 AI 모델을 구축할 수 있는 실무 가이드를 제공합니다.

#### 1. "합성 데이터 병목 현상" 해결
현재 많은 기업들이 LLM을 고도화하기 위해 수십만 건의 합성 데이터를 무작위로 생성하여 GPU 비용을 낭비하고 있습니다. CRAFT는 모델의 약점을 정밀 진단하므로, **단 1,000개의 고품질 표적 데이터만으로도 대규모 비표적 데이터 학습보다 높은 도메인 강화 효과**를 낼 수 있습니다. 이는 기업의 GPU 비용 및 데이터 정제 인건비를 획기적으로 아껴줍니다.

#### 2. 엔터프라이즈 실무 적용 시나리오: 법률 테크(Legal-Tech) 스타트업의 예시
법률 특화 LLM 서비스를 제공하는 기업이 CRAFT를 어떻게 도입할 수 있을까요?

1. **기존 데이터 수집**: 사내 전문 변호사 그룹이 작성한 판례 요약 자료와 함께 구체적인 평가 루브릭("피고인의 감경 사유가 판결문에 명시되어 있는가?", "형법 제XX조 인용이 올바른가?")을 준비합니다.
2. **CRAFT 시스템 적용**: 법률 전용 루브릭을 기반으로 '역량 계층 트리'를 빌드합니다.
3. **약점 자동 진단**: 개발 중인 자사 법률 AI 모델을 돌려봅니다. 시스템이 자동으로 **"우리 모델은 단순 조문 인용은 95% 성공하지만, '교차 국가 간 조세 조약 우회 행위 해석' 노드에서는 통과율이 15%에 불과하다"**는 정밀 보고서를 발행합니다.
4. **저격수식 데이터 보완**: 진단된 약점 노드 컨텍스트를 주입하여 해당 조세 조약 우회 관련 가상 시나리오 및 모범 답안 데이터셋 500개를 긴급 생성하고 모델을 미세조정(SFT)합니다.
5. **결과**: 적은 리소스로 신속하게 법률 전문가 수준의 특화 모델을 배포하는 데 성공합니다.
