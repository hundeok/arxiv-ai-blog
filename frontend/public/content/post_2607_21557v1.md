# OpenForgeRL: 모든 환경에서 하니스 네이티브 에이전트 학습

> **원본 논문 정보**
> - **제목:** OpenForgeRL: Train Harness-native Agents in Any Environment
> - **저자:** Xiao Yu, Baolin Peng, Ruize Xu, Hao Zou, Qianhui Wu, Hao Cheng, Wenlin Yao, Nikhil Singh, Zhou Yu, Jianfeng Gao
> - **발행일:** 2026-07-23
> - **PDF 링크:** [OpenForgeRL: Train Harness-native Agents in Any Environment](https://arxiv.org/pdf/2607.21557v1)


## ✨ 3줄 핵심 요약
- **범용적 에이전트 학습 프레임워크:** 복잡한 인프라 제약 없이 다양한 하니스(Harness)와 환경에서 에이전트를 학습시킬 수 있는 오픈소스 프레임워크입니다.
- **학습과 추론의 분리:** 경량 프록시와 컨테이너 기반 오케스트레이터를 통해 하니스의 모델 호출을 학습 데이터로 전환하여 표준 RL 코드베이스와 결합합니다.
- **성능 검증:** ClawEval, OSWorld 등 다양한 벤치마크에서 기존 오픈소스 모델들을 능가하는 성능을 보였으며, 에이전트의 신뢰성과 다단계 계획 능력을 향상했습니다.

## 📖 쉽게 풀어쓴 1분 핵심
현대의 AI 에이전트는 복잡한 '하니스(인터페이스/도구 모음)'를 통해 동작하지만, 이 하니스들이 너무 복잡해 에이전트를 끝에서 끝까지(End-to-End) 학습시키기 어려웠습니다. **OpenForgeRL**은 에이전트가 사용하는 실제 환경과 하니스를 건드리지 않고, 그 사이에서 데이터를 수집하는 '가벼운 대리자(Proxy)'와 이를 대규모로 실행할 '컨테이너 관리자'를 도입했습니다. 이를 통해 연구자들은 복잡한 설정 없이도 원하는 환경에서 AI 에이전트를 학습시키고 성능을 높일 수 있게 되었습니다.

## 🎯 배경 및 문제점
- **학습의 복잡성:** Claude Code나 OpenClaw 같은 현대적 하니스는 다단계 추론과 도구 사용을 지원하지만, 그 구조가 복잡해 기존의 SFT(지도 미세 조정)나 RL(강화 학습) 스택과 호환되지 않습니다.
- **훈련-배포 격차:** 에이전트가 실제 배포되는 환경과 학습 환경 간의 차이로 인해 최적화가 어려웠습니다.
- **오픈 인프라의 한계:** 상태가 존재하고 멀티 프로세스로 동작하는 하니스 환경을 네이티브하게 표현할 수 있는 표준화된 학습 기반이 부족했습니다.

## 💡 핵심 기술 및 아키텍처
- **경량 프록시(Lightweight Proxy):** 하니스의 모델 호출을 중간에서 가로채 학습 데이터로 기록하는 동시에, 기존 RL 코드베이스(veRL 등)와 연동합니다.
- **컨테이너 기반 오케스트레이터:** 각 에이전트 실행 단위(Rollout)를 원격 컨테이너에서 독립적으로 수행함으로써, 환경 확장이 용이하고 훈련 코드와 배포 코드를 완벽히 분리했습니다.
- **범용성:** 특정 하니스에 종속되지 않으며, 도구 기반 에이전트부터 GUI 브라우저 사용 에이전트까지 다양한 환경에 적용 가능합니다.

## 📊 실험 결과 분석
- **성능 수치:** 
    - **OpenForge-Claw:** ClawEval에서 pass@3 기준 55.9, Qwen-ClawBench에서 33.7 기록.
    - **OpenForge-GUI:** OSWorld-Verified에서 37.7, Online-Mind2Web에서 63.0, WebVoyager에서 72.3 달성.
- **주요 발견:** 
    - RL은 에이전트의 자체 검증, 도구 활용 범위 확대, 다단계 계획 완수와 같은 '신뢰성' 측면을 크게 개선함.
    - 다만, '오류 복구(Error Recovery)' 능력은 여전히 낮은 수준으로, 이는 추가적인 전문 데이터 학습이 필요함을 시사함.
    - 일부 하니스는 학습 난이도가 현저히 높음이 확인됨.

## 🌍 세상에 미치는 영향 & 실무 적용 사례
- **연구 가속화:** 하니스 기반 에이전트를 연구하려는 학계 및 산업계 연구자들이 실제 배포 환경과 동일한 조건에서 에이전트를 직접 학습시킬 수 있는 길을 열었습니다.
- **실무 적용:** 웹 브라우징 자동화, 도구 기반 워크플로우 수행 등 다양한 분야의 에이전트 성능을 실질적으로 높일 수 있으며, 오픈소스 코드와 데이터 공개를 통해 에이전트 학습 생태계의 장벽을 낮출 것으로 기대됩니다.
