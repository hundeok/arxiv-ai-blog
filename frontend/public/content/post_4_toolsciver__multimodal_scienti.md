# 🔬 AI가 논문을 읽고 팩트체크를 하는 법: ToolSciVer 기술 분석

* **저자**: Binglin Zhou, Peng Shi, Ryo Kamoi, Nan Zhang, Rui Zhang (펜실베이니아 주립대학교, 워털루 대학교)

### ✨ [3줄 핵심 요약]
1. **목표**: 텍스트, 표, 차트, 복잡한 다중 패널 그림이 혼재된 과학 논문에서 특정 주장(Claim)의 진위 여부를 정확하게 검증하는 것.
2. **해결책**: VLM(비전-언어 모델)이 스스로 표의 특정 행/열을 추출(`Table Focus`), 차트를 구조화된 데이터로 변환(`Chart Parse`), 고해상도 확대(`Region Zoom`)를 수행할 수 있는 '타입 기반 비주얼 툴'을 부여함.
3. **학습 방법**: 강화학습 알고리즘인 **GRPO(Group Relative Policy Optimization)**를 사용해 불필요한 도구 사용은 줄이고(효율성), 정확하고 올바른 형식의 도구 호출을 수행하도록 최적화함.

### 📖 [쉽게 풀어쓴 1분 핵심]
> **💡 한 줄 요약: 돋보기와 스마트 스캐너를 쥐여주고 "꼭 필요할 때만 써서 팩트체크해!"라고 강화훈련을 시킨 AI 검증 요원**

복잡한 물리, 화학, 생물학 논문이 가득 쌓인 도서관이 있다고 상상해 보세요. 누군가 어떤 과학적 주장을 들고 왔을 때, 이를 검증하려면 논문 구석에 있는 아주 작은 표와 복잡한 꺾은선그래프를 꼼꼼하게 읽어야 합니다. 

기존의 눈이 침침한 거대 AI 모델(VLM)들은 이 복잡한 이미지들을 멀리서 대충 훑어보고 "음, 대충 맞는 것 같네요!"라고 짐작해 버리곤 했습니다. 당연히 오답률이 높았죠.

**ToolSciVer**는 이 AI에게 아주 강력한 **'탐정 도구 세트'**를 선물했습니다.
1. **표 스캐너(Table Focus)**: 복잡한 표에서 내가 원하는 '4번째 줄'만 쏙 뽑아서 읽어줍니다.
2. **그래프 변환기(Chart Parse)**: 이리저리 꼬인 꺾은선그래프를 깔끔한 숫자 목록(`[시간: 1초, 값: 10], [시간: 2초, 값: 20]...`)으로 번역해 줍니다.
3. **슈퍼 돋보기(Region Zoom)**: 복잡한 세포 분열 그림에서 내가 원하는 특정 구역만 아주 선명하게 확대해서 보여줍니다.

그리고 AI가 도구를 너무 남용해서 전기세를 낭비하거나 엉뚱한 도구를 쓰지 않도록, 엄격한 코치(GRPO 강화학습)가 옆에서 **"진짜 필요할 때만 딱 맞게 도구를 써서 정확하게 맞춰!"**라며 점수(보상)를 주며 훈련시켰습니다. 그 결과, AI는 최소한의 도구만 똑똑하게 사용하여 어려운 과학적 사실을 완벽하게 검증하는 일류 과학 탐정으로 거듭나게 되었습니다.

### 🎯 [배경 및 문제점]
> **💡 한 줄 요약: 고해상도 구조화된 시각 자료(표, 차트) 앞에서는 GPT-4o 같은 빅테크 모델들도 여전히 헤매고 있습니다.**

현대 과학 분야의 사실 검증(Scientific Claim Verification)은 단순히 텍스트만 팩트체크하는 수준을 넘어섰습니다. 이제는 논문의 그림(Figure), 표(Table), 차트(Chart)와 본문을 유기적으로 통합하여 분석하는 **멀티모달 과학 주장 검증(MSCV)** 능력이 필수적입니다.

#### 기존 연구 및 상용 모델의 명확한 한계점
1. **미세한 시각 증거 정보 유실 (Information Loss)**:
   대부분의 비전-언어 모델(VLM)은 이미지를 고정된 해상도로 다운샘플링하여 입력받습니다. 이 과정에서 수십 개의 행과 열로 이루어진 미세한 논문 표나 다중 패널(Multi-panel) 구조의 실험 장치 도식 내 세부 텍스트들이 뭉개져 식별 불가능한 상태가 됩니다.
2. **차트 및 표 구조 분석의 부재 (Structured Reasoning Failure)**:
   GPT-4o, Claude 3.5 Sonnet 같은 강력한 상용 모델조차 복잡한 그래프의 축(Axis) 레이블을 잘못 읽거나 서로 다른 두 데이터 시리즈의 수치를 혼동하는 문제를 자주 보입니다.
3. **일반 도구 사용 모델의 한계 (Generic Tool Limitations)**:
   기존의 비주얼 에이전트 도구들(예: ViperGPT, VTool-R1 등)은 단순히 이미지를 자르거나(Crop), 세그멘테이션을 하거나, 단순 OCR을 처리하는 데 그칩니다. 과학 논문의 데이터 속성(행/열 비교, 차트 트렌드 파악 등)에 최적화된 **'Task-Specific Evidence Extractor'**가 없어 MSCV 환경에서 효율성이 크게 떨어집니다.

### 💡 [핵심 기술 및 아키텍처]
> **💡 한 줄 요약: 세 가지 특화 도구 인터페이스와 GRPO 알고리즘 기반의 다중 보상 체계 설계가 핵심입니다.**

ToolSciVer는 복잡한 수작업 전처리 없이, 모델이 스스로 필요에 따라 최적의 비주얼 도구를 호출하여 증거를 취합하는 **'액티브 에비던스 애퀴지션(Active Evidence Acquisition)'** 패러다임을 제안합니다.

```
[Claim / Context Input] 
       │
       ▼
 [Policy VLM (Qwen/InternVL/Gemma)] <───┐
       │ (Step 1: Do I need a tool?)    │
       ├─────────────────┐              │ (Feedback & Observations)
       │ Yes             │ No           │
       ▼                 ▼              │
[Select Tool Class]  [Direct Answer]    │
 1. Table Focus                         │
 2. Chart Parse ────────────────────────┘
 3. Region Zoom
```

#### 1. 타입 인지형 비주얼 도구 세트 (Type-Aware Visual Tool Suite)

* **Table Focus (`focus_row`, `focus_column`)**  
  논문 내 표 이미지가 입력되면, 1차적으로 OCR을 백엔드에서 작동시켜 경량화된 CSV 형태로 래핑합니다. 모델은 특정 행 번호나 열 제목을 인자(`row_number`, `column_number`)로 호출하여, 전체 이미지 대신 **주장 검증에 필요한 타깃 행/열의 데이터만 텍스트 증거(Grounded Text)로 수집**합니다. 이를 통해 쓸데없는 배경 노이즈 데이터를 줄이고 컨텍스트 윈도우를 확보합니다.
  
* **Chart Parse (`parse_content`)**  
  꺾은선그래프, 바 차트, 산점도 등의 시각적 경향성을 인지하기 위해 차트 해석 전용 서브 모델을 백엔드로 활용합니다. 이미지 경로를 인자로 호출하면 해당 그래프 내의 좌표값, 범례(Legend), 그리고 이를 매핑한 **JSON 형식의 구조화된 데이터(JSON-like key-value entries)**를 반환합니다.
  
* **Region Zoom (`image_zoom_in`)**  
  다중 패널 생물학 현미경 사진이나 복잡한 물리 도식에 사용됩니다. 화면의 절대 해상도에 의존하지 않도록 `[0, 1000]` 범위로 정규화된 좌표계(`x1, y1, x2, y2`)를 입력하면, 해당 영역을 고해상도로 크롭 및 업샘플링하여 다시 입력 이미지 형태로 제공합니다.

#### 2. GRPO(Group Relative Policy Optimization)를 통한 selective tool use 학습

이 모델은 단순히 프롬프트 엔지니어링으로 작동하는 것이 아닙니다. 도구 호출 메커니즘을 온디바이스에서 효율적으로 학습시키기 위해, DeepSeek-Math 등에서 효용성이 입증된 **GRPO 기반 강화학습**을 사용합니다. 

GRPO는 개별 액션마다 가치 함수(Value Function) 모델을 따로 두지 않고, 하나의 질문에 대해 $G$개의 롤아웃(Rollout)을 샘플링한 뒤 이들의 상대적 보상 평균과 표준편차를 이용해 어드밴티지($A_i$)를 계산함으로써 메모리 효율성과 학습 안정성을 동시에 극대화합니다.

$$A_i = \frac{R_i - \text{mean}(R_G)}{\text{std}(R_G)}$$

#### 3. 다중 보상 함수 설계 (Composite Reward Design)
ToolSciVer 학습을 관통하는 가장 핵심적인 기여는 바로 정교한 보상 공식($R(\tau)$)에 있습니다.

$$R(\tau) = s_{\text{eff}}(\tau) \left( r_{\text{ans}}(\tau) + \lambda_{\text{fmt}} r_{\text{fmt}}(\tau) \right) + \lambda_{\text{len}} r_{\text{len}}(\tau) + \lambda_{\text{tool}} r_{\text{tool}}(\tau)$$

1. **최종 정답 보상 ($r_{\text{ans}}$)**: 예측 정답 여부에 따른 바이너리 보상 ($0$ 또는 $1$).
2. **형식 준수 보상 ($r_{\text{fmt}}$)**: CoT 생각 흐름 태그와 최종 정답 포맷 마커를 안정적으로 출력했는지 판별 ($0.5 + 0.5$ 구조).
3. **길이 페널티 보상 ($r_{\text{len}}$)**: 모델이 쓸데없이 긴 문장을 생성해 계산 비용을 낭비하지 않도록 소프트 스레숄드($L_{\text{soft}}$) 이상 구간부터 선형적인 페널티를 부과합니다.
4. **도구 효율성 계수 ($s_{\text{eff}}$)**: 이 연구의 독창적인 파트입니다. 정답을 맞힌 롤아웃 그룹 중 **가장 최소한의 도구 호출 수($n^*$)**를 기준 삼아, 불필요하게 도구를 많이 쓴 롤아웃에는 사인/코사인 주기 함수 형태의 페널티를 주어 선택적 도구 호출(Selective Tool Use)을 유도합니다.
5. **도구 상호작용 페널티 ($r_{\text{tool}}$)**: API 호출 파싱 오류(Malformed turn), 잘못된 좌표 인자 전달(Execution failure) 등의 문제 발생 시 세밀하고 즉각적인 마이너스 피드백을 주어 에이전트의 오작동율을 0%에 수렴하게 만듭니다.

### 📊 [실험 결과 분석]
> **💡 한 줄 요약: ToolSciVer는 오픈소스 VLM의 체급을 뛰어넘어, 유료 상용 모델 수준의 MSCV 검증률을 달성했습니다.**

#### 1. 종합 성능 비교 (SCIVER & MuSciClaims)
본 연구에서는 **Qwen3.5, InternVL3.5, Gemma4** 계열 모델을 대상으로 성능을 측정했습니다.

* **SCIVER** (종합 과학 검증 벤치마크) & **MuSciClaims** (그림 중심 과학 검증 벤치마크) 결과:

| 모델 (Backbone) | 방법론 (Method) | SCIVER Overall Acc (%) | MuSciClaims Overall Acc (%) |
| :--- | :--- | :---: | :---: |
| **Qwen3.5-4B** | COT (No Tool) | 70.10 | 60.99 |
| | Tool Prompt (No RL) | 70.10 | 71.88 |
| | VTOOL-R1 (RL Baseline) | 81.20 | 76.83 |
| | OPENTHINKIMG (RL Baseline) | 75.80 | 76.04 |
| | **ToolSciVer (Ours)** | **81.20** | **78.61** |
| **InternVL3.5-8B**| COT (No Tool) | 67.44 | 68.91 |
| | Tool Prompt (No RL) | 60.00 | 67.72 |
| | **ToolSciVer (Ours)** | **76.40** | **74.85** |
| *Proprietary* | GPT-4o COT | 73.89 | 73.27 |
| *Proprietary* | Claude Sonnet 4.6 COT | 70.50 | 79.41 |

* **결과 해석**: 
  - **ToolSciVer**는 단순 CoT 대비 성능을 대폭 상승시켰습니다. 특히 정보 밀도가 매우 높은 `MuSciClaims` 데이터셋에서 **Qwen3.5-4B 기반 ToolSciVer는 78.61%를 기록하며, 훨씬 더 거대한 상용 모델인 GPT-4o(73.27%)의 성능을 추월**했습니다.
  - 도구 학습 과정 없이 프롬프트만 사용한 `Tool Prompt` 방법론의 경우, 모델이 복잡한 인자 파싱에 실패하거나 엉뚱한 순간에 도구를 남용하여 오히려 성능이 떨어지는 극심한 불일치(Inconsistency) 현상을 보였습니다. 이는 **강화학습을 통한 Selective Tool-use 훈련이 필수적임**을 완벽하게 입증합니다.

#### 2. 왜 더 잘할까? 세부 진단 결과 분석 (Diagnostic Study)

* **REAR (Relevant Evidence Acquisition Rate, 증거 획득 성공률)**:
  논문 검증을 위해 반드시 읽었어야 하는 핵심 도표 데이터를 모델이 성공적으로 획득했는지 정밀 평가한 결과입니다.

| 방법론 | Table 영역 수집 (%) | Chart 영역 수집 (%) | Multi-panel 수집 (%) | 전체 평균 (%) |
| :--- | :---: | :---: | :---: | :---: |
| COT (No Tool) | 93.00 | 87.00 | 69.00 | 83.00 |
| Tool Prompt | 87.00 | 88.00 | 75.00 | 83.33 |
| **Ours (ToolSciVer)** | **98.00** | **93.00** | **79.00** | **90.00** |

* **해석**: ToolSciVer는 멀티 패널 이미지와 복잡한 차트 영역에서 기존 대비 압도적인 증거 수집 성공률(평균 90.00%)을 기록했습니다. 이는 잘못된 증거를 기반으로 환각(Hallucination)을 일으키는 비율을 정량적으로 예방했음을 증명합니다.

### 🌍 [세상에 미치는 영향 & 실무 적용 사례]
> **💡 한 줄 요약: 리서치, 의료, 금융 등 고밀도 원본 문서 검증이 필요한 도메인에서 당장 적용할 수 있는 설계 패턴입니다.**

ToolSciVer가 제시한 기술 아키텍처는 비단 학술 연구에만 국한되지 않습니다. 엔지니어와 기업 비즈니스 관점에서 이를 어떻게 활용할 수 있을까요?

#### 1. 실무 도입 시나리오: "스마트 금융 공시 및 감사 자동화 시스템"
* **기존 문제**: 수십 페이지 분량의 기업 분기 보고서(PDF)에는 수많은 표와 실적 차트가 담겨 있습니다. "올해 영업 이익 성장률이 업계 평균을 상회한다"는 주장을 감사 AI가 검증하려면 오차가 있어서는 안 됩니다.
* **ToolSciVer 패턴 적용**:
  * 회계 감사용 VLM에 특정 기업의 재무제표 표를 행/열 단위로 스캐닝하는 `Financial Table Focus` API를 바인딩합니다.
  * 추이 그래프 해석을 위해 Matplotlib이나 PDF 내 벡터 차트 데이터를 JSON으로 덤프하는 `Financial Chart Parser`를 융합합니다.
  * AI가 쓸데없이 고비용 API를 남용하지 않도록 **GRPO 효율성 보상 알고리즘**을 금융 도메인 데이터로 가볍게 파인튜닝(SFT + RL)하여 온프레미스로 서빙합니다.

#### 2. 실무 개발자를 위한 멘토의 팁 (Implementation Tip)
* **Q. 상용 API 요금이 걱정되는데, 효율성 보상 공식을 어떻게 활용하나요?**  
  현업에서 비주얼 에이전트를 프로덕션에 배포할 때 가장 큰 장애물은 **"모델이 불필요한 도구를 루프 돌며 계속 호출하는 비용 문제"**입니다. 이 논문에서 제시한 **OTC 스타일의 효율성 계수($s_{\text{eff}}$)** 보상 설계를 차용하세요. 
  "최소한의 API 호출로 유효한 정답을 낸 에이전트 경로에 강력한 상대 보상을 주고, 과도한 API 호출에는 페널티를 주는 방식"은 강화학습 과정에서 모델이 자연스럽게 **최소한의 자원으로 문제를 해결하려는 자린고비식 지능**을 학습하게 만듭니다.

* **Q. 도구 호출 오류가 빈번합니다.**  
  논문의 `Dense Tool-Interaction Penalty` 공식을 참고하여, 실패에 대한 피드백을 세분화(포맷 실패 -0.2, 실행 에러 -0.3, 에러 후 비정상 종료 -0.4 등)해야 합니다. 실패를 하나의 뭉뚱그린 마이너스 점수로 주면 강화학습의 그라디언트 노름(Gradient Norm)이 튀어 학습이 무너지기 쉽습니다. 세밀하게 쪼갠 페널티가 강화학습의 안정성을 보장합니다.
