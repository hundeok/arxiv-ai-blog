# 작고 자유로우며 효과적인 악성코드 분석: 단일 거대언어모델을 능가하는 오픈 웨이트 소형언어모델 오케스트레이션

## ✨ 3줄 핵심 요약
* 악성코드 분석 현장에서 단일 거대언어모델(LLM)의 비용과 불투명성을 극복하기 위해 소형언어모델(SLM)들을 조합한 오케스트레이션 아키텍처를 설계함.
* 4가지 오케스트레이션 구조를 실험한 결과, 하이브리드 모델(Qwen3-4B + Foundation-Sec-8B)이 최고 성능의 전문 보안 모델과 범용 LLM의 성능을 상회함.
* 데이터 기반 오케스트레이션이 소형 모델의 추론 오류를 교정하고 실제 악성코드 분석 성능을 크게 향상시킬 수 있음을 입증함.

## 📖 쉽게 풀어쓴 1분 핵심
악성코드 분석은 방대한 보고서를 빠르게 해석해야 하는 어려운 작업입니다. 기존의 거대언어모델(LLM)은 성능은 좋지만, 사용 비용이 비싸고 내부가 공개되지 않아 보안 현장에서 신뢰하기 어렵습니다. 본 연구는 성능은 낮지만 가볍고 공개된 소형언어모델(SLM) 여러 개를 마치 전문가 팀처럼 협업시키는 방법을 찾았습니다. 단순히 모델을 합치는 것을 넘어, 서로 토론하고 증거를 확인하는 방식을 적용하자, 거대한 LLM보다 더 정확하게 악성코드의 위협을 판단하고 추론 오류를 스스로 수정하는 결과를 보였습니다.

## 🎯 배경 및 문제점
* **비용과 투명성:** 폐쇄형 LLM은 사용 비용이 높고 내부 작동 원리가 불투명하여 보안 분석 시 데이터 프라이버시와 의사결정의 재현성에 문제가 발생함.
* **리소스 제약:** 대형 오픈 웨이트 모델은 운영에 막대한 컴퓨팅 자원이 필요하여 리소스가 제한된 현장 배포가 어려움.
* **추론 한계:** 개별 SLM은 복잡한 보안 관련 추론 작업에서 LLM보다 성능이 떨어지는 경향이 있음.

## 💡 핵심 기술 및 아키텍처
연구팀은 단일 GPU(RTX 4090, 4비트 양자화 기준) 환경에서 다음 4가지 오케스트레이션 구조를 설계함.
1. **에이전트 파이프라인:** 분석을 증거 수집과 추론 단계로 분리.
2. **적대적 토론:** 두 에이전트가 서로의 추론을 반복적으로 비판.
3. **계층적 컨설팅:** 범용 SLM이 보안 전문 모델에게 자문을 구함.
4. **하이브리드 아키텍처:** 증거 기반 파이프라인과 적대적 토론을 결합.

## 📊 실험 결과 분석
* **하이브리드 시스템 성능:** Qwen3-4B와 Foundation-Sec-8B를 결합한 하이브리드 모델이 35.30%의 정확도를 기록함. 이는 가장 뛰어난 보안 전문 모델(22.54%) 및 증거 기반이 없는 최상위 범용 모델(34.77%)의 성능을 상회함.
* **비교 수치:** 증거를 기반으로 했을 때 Gemini 모델이 38.22%로 가장 높은 성능을 보였으나, 소형 모델 조합인 하이브리드 시스템이 그 격차를 상당히 좁힘.
* **정성적 평가:** EtherHiding 및 ClickFix와 같은 최신 우회 기법 분석 시, 하이브리드 시스템이 단일 모델이 범하는 추론 오류를 효과적으로 교정함을 확인.

## 🌍 세상에 미치는 영향 & 실무 적용 사례
* **실무 적용:** 기업은 고가의 LLM API 비용 없이도 현장(on-premise)에서 보안 분석 시스템을 직접 구축하고 운영할 수 있음.
* **보안 신뢰성 강화:** 오픈 웨이트 모델을 사용하므로 모델 가중치를 감사할 수 있고, 자체 정책에 따른 위험 통제가 가능함.
* **기술적 가치:** 증거 기반 오케스트레이션이 소형 모델들의 협업 효율을 극대화하여, 특정 도메인(악성코드 분석)에서 거대 모델의 대안이 될 수 있음을 증명함.
